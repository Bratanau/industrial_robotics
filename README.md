## Реализованные компоненты

### 1. Управление и движение

#### `TrackController.cs`
Контроллер гусеничного робота с дифференциальным приводом:
- **Дифференциальное управление** — смешивание газа и руля в скорости левого/правого борта.
- **Моделирование моторов PWM** — мёртвая зона, минимальный стартовый порог, масштабирование м/с → PWM.
- **Плавный разгон** через `Rigidbody.MovePosition/MoveRotation` в `FixedUpdate`.
- **Стабильная физика** — `RigidbodyInterpolation.Interpolate`, `Continuous` collision detection, демпфирование.
- **Входные интерфейсы** — `SetCommand(gas, steer)` для нейросети/ROS, клавиатура WASD для отладки.
- **Динамическая калибровка** — `SetMotorParams()` для Domain Randomization.

### 2. Сенсоры

#### `VirtualSensors.cs`
Симуляция физических датчиков через `Physics.Raycast`:

| Датчик | Описание |
|---|---|
| **Ультразвуковые (×2)** | Левый и правый УЗ. Веер из 5 лучей в конусе 30°, макс. дистанция 2м. Игнорирует мяч (реальный УЗ не видит маленький мяч). |
| **ИК препятствий (×3)** | Лево / центр / право. Короткий луч (15 см), бинарный: 1 = стена близко, 0 = пусто. |
| **ИК клешни** | Луч (8 см) внутрь захвата. Реагирует **только на мяч** — определяет факт захвата. |

- Визуализация лучей через `Gizmos` в редакторе.
- Настраиваемые параметры: углы обзора, дальность, количество лучей, маски слоёв.

#### `GripperController.cs`
Контроллер клешни:
- **Логический захват** — при обнаружении мяча ИК-сенсором клешни, мяч переключается в kinematic и привязывается к точке `HoldPoint` между губками.
- Физика мяча отключается на время удержания (решение проблемы проскальзывания в физике Unity).
- `IsHolding` — публичное состояние удержания мяча.

### 3. Нейросетевой мозг (`RobotBrain`)

**`RobotBrain.cs`** (v6.1) — центральный агент, наследник `Unity.MLAgents.Agent`. 913 строк, 7 модулей наград, 18 наблюдений, 3 непрерывных действия.

#### Миссия
> Ехать **прямо** на мяч, подъехать **медленно**, чтобы мяч оказался **между губками** клешни, и **остановиться**, не толкая его.

#### Ключевые архитектурные решения

- **Пропорциональная награда за сближение** (v5.0) — вместо бинарной награды, которая вызывала дробь на месте.
- **Двойные ворота центровки** (v5.3) — мяч по центру кадра **И** камера не отвёрнута сервоприводом. Иначе робот ехал боком.
- **Штраф за сдвиг мяча** (v6.0) — вместо штрафа за касание, агент учится аккуратно подходить.
- **Фикс зависания эпизодов** (v6.1) — лимит эпизода проверяется первым, защёлка работает только в инференсе.

#### Режимы работы

| Режим | Описание |
|---|---|
| **Training (Default)** | Обучение с PPO, случайные препятствия, награды вычисляются |
| **Heuristic** | Ручное управление WASD + автоматическая обходная логика |
| **Inference Only** | Прогон обученной .onnx модели в реальном времени |
| **Mission Mode** | Демо-инференс на железе — отключает телепорты и `EndEpisode` |

### 4. Система наград

Реализована 7-компонентная экономика наград:

| Сигнал | Тип | Описание |
|---|---|---|
| **R1 — Approach** | Пропорциональная | Delta дистанции × масштаб × множитель близости × фактор центровки. Чем ближе мяч — тем выше ставка. |
| **R2 — Success** | Ступенчатая | Касание (0.5) → удержание стоя (0.05/шаг) → полный успех (5.0) |
| **R3 — Go Straight** | Бонусы/штрафы | Двойные ворота центровки, бонус за медленный подъезд, штраф за быстрый, бонус за ползание в слепой зоне, следование подсказке LiDAR |
| **R4 — Sensor Penalties** | Штрафы | Градиентный штраф УЗ-датчиков, бинарный штраф ИК-препятствий |
| **R5 — Smoothness** | Штраф action-rate | Квадратичный штраф резких изменений команд (газ/руль/камера) |
| **R6 — Anti-Snowplow** | Штраф за сдвиг | Штраф пропорционален скорости мяча при контакте (не за само касание!) |
| **R7 — Timeout/Stuck** | Штрафы | Таймаут эпизода, штраф за застревание (200 шагов без перемещения при газе > 0.15), штраф за ползание (gas < 0.15) |

#### Подсказка от LiDAR (Practice 6)
- Симуляция векторной подсказки от второго робота/лидара.
- На каждом эпизоде с вероятностью `hintDropoutChance` захватывается угол и дистанция до мяча.
- Доверие (`hintConfidence`) затухает со временем и при движении/повороте робота.
- Агент получает бонус за движение вперёд вдоль вектора подсказки, пока доверие живо.

### 5. Обучение с подкреплением

#### Конфигурация обучения
- **Тренер**: PPO (Proximal Policy Optimization)
- **Сеть**: 2 скрытых слоя × 256 нейронов, LSTM с памятью 256, последовательность 64
- **Batch size**: 2048, **Buffer size**: 40960
- **Learning rate**: 0.00025 (линейный scheduling)
- **Max steps**: 50M (config.yaml) / 40M (curriculum)
- **Time horizon**: 1000 шагов

#### `ObstacleRandomizer.cs`
- Рандомизация расположения препятствий на каждом эпизоде.
- Режимы: **фиксированные** (режим 0) и **случайные** (режим 1) — управляется через curriculum из YAML.
- Правила размещения: минимальное расстояние между препятствиями, отступ от стен, случайный поворот.

### 6. Визуальное восприятие

#### `SimulatedYoloCamera.cs`
Симуляция бортовой YOLO-камеры без рендеринга:
- Проецирование 3D-позиции мяча в 2D viewport.
- Проверка FOV (горизонтальный угол 40°), максимальной дальности (2 м).
- **Line of Sight** — RaycastAll с пропуском собственных коллайдеров робота.
- Выход: `IsVisible`, `RelativeAngle` (−1..1), `NormalizedDistance` (0 = близко, 1 = далеко).

#### `RealVision.cs` (Practice 7 — HIL)
Приём реальных данных YOLO с бортового ПК по UDP:
- UDP-листенер на порту 5005, JSON-пакеты от `yolo_vision_node.py`.
- **Инверсия и калибровка дистанции** — конвертация `bboxHeight/frameHeight` в сим-конвенцию (0 = близко, 1 = далеко).
- Watchdog-таймаут: если пакеты прекратились — мяч считается потерянным.
- Прозрачная замена симулированной камеры на реальную через флаг `useYOLO`.

### 7. Sim2Real и Domain Randomization

#### `DomainRandomizer.cs`
Полный набор рандомизаций (4 уровня):

| Уровень | Что рандомизируется | Когда |
|---|---|---|
| **1. Физика** | Масса робота (1–4 кг), скорости моторов, масса и масштаб мяча | Каждый эпизод |
| **2. Шум УЗ** | Белый шум ±5% к показаниям ультразвуковых датчиков | Каждое наблюдение |
| **3. YOLO dropout** | Потеря визуального контакта при быстром вращении (5–16 шагов) | Каждое наблюдение |
| **4. Задержка команд** | FIFO-очередь с задержкой 8–14 шагов (160–260 мс) | Каждое действие |

- **Master switch** — один чекбокс для включения/выключения всей рандомизации.
- Автоматическое включение шума датчиков только на финальной фазе Sim2Real (через `sim2real_noise` из YAML).
- Задержка команд включена и в инференсе — политика обучена компенсировать её.

### 8. ROS-интеграция

#### `ROSBridge.cs`
Мост между ML-Agents инференсом и реальным роботом:
- Публикация в топики `/cmd_vel` (Twist), `/cmd_gripper` (Int32), `/cmd_camera_pan` (Float32).
- **EMA-сглаживание** (alpha=0.8) — убирает высокочастотное дрожание сигнала, защищая моторы и редуктор.
- **Watchdog fail-safe** — при отсутствии команд 0.5с автоматически отправляет STOP.
- Лимиты скорости: линейная (0.5 м/с), угловая (1.0 рад/с).

### 9. Отладка и диагностика

#### `BrainDebugHUD.cs`
On-screen HUD, показывающий в реальном времени:
- Все наблюдения (ультразвук, ИК, угол/дистанция мяча, серво, подсказка LiDAR).
- Текущие действия (газ, руль, камера).
- Пошаговую и накопленную награду с цветовой индикацией.

#### `DiagnosticLogger.cs`
CSV-логгер для разбора Sim-to-Real расхождений:
- Запись на каждый шаг принятия решения: зрение, физические сенсоры, действия ИИ, одометрия.
- 20 колонок, ограничение размера файла (2000 строк по умолчанию).

#### `SizeInspector.cs`
Утилита для проверки размеров объектов в метрах через `Gizmos`.

## Кривая обучения (Curriculum Learning)

Реализована 7-ступенчатая кривая (`config_curriculum.yaml`), управляющая тремя параметрами одновременно:

| Фаза | Дистанция мяча | Препятствия | Sim2Real шум | Длина эпизода |
|---|---|---|---|---|
| **Phase0** | 1.2 м (близко) | Фиксированные | Выкл | 1500 шагов |
| **Phase1** | 1.2 м (близко) | Случайные | Выкл | 1500 шагов |
| **Phase2** | 1.9 м (средне) | Фиксированные | Выкл | 1800 шагов |
| **Phase3** | 1.9 м (средне) | Случайные | Выкл | 1800 шагов |
| **Phase4** | 2.6 м (далеко) | Фиксированные | Выкл | 2400 шагов |
| **Phase5** | 2.6 м (далеко) | Случайные | Выкл | 2400 шагов |
| **Phase6** | 2.6 м (далеко) | Случайные | **Вкл** | 2400 шагов |

Переход между фазами происходит при достижении порога средней награды (5.0 → 5.5 → 6.0), минимум 400 эпизодов на фазу.

## Как запустить

### Для ручного управления (симуляция)

1. **Unity 6** (6000.0.23f1) через Unity Hub.
2. Открыть проект → `Assets/Scenes/SampleScene.unity`.
3. Нажать **Play**.

### Для обучения (ML-Agents)

1. Перейти в ветку `wexler-rl`.
2. Открыть `Assets/Scenes/SceneW.unity`.
3. Собрать Linux-билд → запустить `fix_build.sh`.
4. `mlagents-learn config.yaml --run-id=<id>`.
5. Нажать **Play** в Unity или запустить Linux-билд.
6. Мониторинг через TensorBoard: `tensorboard --logdir results`.

### Для инференса обученной модели

1. Поместить `.onnx` модель в `Assets/ModelsW/`.
2. В Prefab Mode `TrainingAreaW` → robot → Behavior Parameters:
   - `Model` = загруженная .onnx
   - `Behavior Type` = **Inference Only**
3. Нажать **Play**.
